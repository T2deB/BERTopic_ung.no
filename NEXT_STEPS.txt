Recommended next steps when returning
-------------------------------------
A) Inspect cluster quality
   1. Open clusters_catalog.csv
      - Focus on: cluster size, top_unigrams, top_bigrams, exemplar_text.
   2. Open clusters_keywords_unigrams.csv and clusters_keywords_bigrams.csv
      - Confirm top terms are coherent within each cluster.

B) Run review packaging
   Command:
   python prepare_cluster_review.py --output-dir "<YOUR_OUTPUT_DIR>"

   Then inspect:
   - cluster_review/cluster_XXXX/sample_for_labeling.csv
   - cluster_review/unclustered/sample_for_review.csv

C) Labeling workflow
   1. Fill cluster_review/labels_template.csv:
      - label_name
      - label_description
      - merge_into_cluster_id (optional)
      - drop_cluster (optional)
   2. Note recurring themes in unclustered rows.
   3. Add those as seed topics in guided_topics.json for the next run.

D) If too many unassigned rows
   - Relax thresholds slightly and rerun clustering:
     - ASSIGN_MIN_SIM: try 0.42 (from 0.44)
     - ASSIGN_MIN_MARGIN: try 0.01 (from 0.015)
   - Keep only one change per run to compare impact clearly.

E) Keep run logs
   - Save clustering_run_params.json per run (copy/rename with date)
   - Keep a short run diary: sample size, thresholds, % unassigned, notes.
